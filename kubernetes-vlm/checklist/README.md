# Чек-лист проекта: VLM Сортировка на складе

Этот чек-лист предназначен для контроля процесса подготовки, развертывания и тестирования системы складской сортировки на базе VLM в Kubernetes.

## 1. Подготовка инфраструктуры и оборудования
- [ ] Установлены и настроены IP-камеры в зоне сортировки (минимум 2 ракурса для избежания слепых зон).
- [ ] Настроен стабильный RTSP-поток с камер (выделенная подсеть или VLAN).
- [ ] Подготовлен Edge-сервер (например, с GPU уровня NVIDIA RTX 4090 или Tesla T4 / A10G) с достаточным объемом VRAM (от 16GB для комфортной работы 3B VLM и YOLO).
- [ ] Операционная система обновлена (Linux/Ubuntu или Windows с настроенным WSL2).

## 2. Настройка Kubernetes кластера (Edge)
- [ ] Установлен легковесный Kubernetes кластер (k3s).
- [ ] Установлены драйверы NVIDIA для хоста.
- [ ] Установлен NVIDIA Device Plugin для Kubernetes (чтобы поды могли запрашивать ресурс `nvidia.com/gpu`).
- [ ] Установлен Helm для удобного управления пакетами.
- [ ] (Опционально) Развернут локальный Docker Registry для хранения кастомных образов.

## 3. Подготовка и сборка контейнеров
- [ ] Собран образ `camera-sync-node` (FFmpeg/OpenCV для захвата RTSP и синхронизации).
- [ ] Собран образ `yolo-detector` (настроен на детекцию коробок, корзин, работников).
- [ ] Скачаны веса модели VLM (например, LFM2-VL-3B) в формате, оптимизированном для инференса (ONNX / TensorRT / vLLM).
- [ ] Собран образ `vlm-inference` (сервер логики и инференса VLM с темпоральным буфером).
- [ ] Собран образ `event-extractor` (интеграция с WMS/Kafka).

## 4. Развертывание в Kubernetes
- [ ] Развернут брокер сообщений (Kafka/Redis) для связи микросервисов.
- [ ] Запущен Deployment YOLO детектора (проверено выделение GPU: `kubectl get pods`).
- [ ] Запущен Deployment VLM сервера (проверено потребление VRAM и готовность API).
- [ ] Запущен Deployment синхронизации камер (проверено, что кадры успешно пишутся в буфер/Redis).
- [ ] Запущен модуль интеграции и настроены ConfigMaps/Secrets (адреса WMS, креды).

## 5. Тестирование и калибровка
- [ ] Проведен тест задержки (Latency): время от физического действия до появления JSON-события в WMS не должно превышать 1-2 секунды.
- [ ] Калибровка ракурсов: VLM успешно распознает зоны на основе переданных кадров (без ложных срабатываний).
- [ ] Тестирование темпоральной памяти: проверка корректности определения события при N=3 кадров.
- [ ] Температура (Temperature) VLM установлена на низкий уровень (0.1 - 0.3) для детерминированности.
- [ ] Проверка на галлюцинации (помогает ли YOLO отсекать выдуманные VLM действия).

## 6. Ввод в эксплуатацию
- [ ] Настроен мониторинг (Prometheus + Grafana) для отслеживания FPS, потребления GPU и ошибок API.
- [ ] Инструкции переданы администраторам и дежурной смене.
- [ ] Настроено автоматическое удаление/ротация логов и сырых кадров, чтобы не переполнить диск.
