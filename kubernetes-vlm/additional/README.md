# Развертывание системы сортировки на базе VLM в Google Cloud (GKE)

В директории `helm` представлены шаблоны для развертывания распределенной системы складской сортировки в облачной среде Google Kubernetes Engine (GKE).

В отличие от локального (Edge) развертывания через k3s, развертывание в GKE актуально, когда вы передаете сжатые потоки видео или экстрагированные кадры с нескольких локальных складов в единый мощный облачный кластер для аналитики.

## Особенности Helm-чарта для GKE

1. **Node Selectors и GPU**:
   В файле `values.yaml` настроены селекторы узлов GKE для привязки подов к конкретным типам видеокарт:
   * Для легковесного YOLO детектора используется недорогая карта **NVIDIA Tesla T4** (`cloud.google.com/gke-accelerator: nvidia-tesla-t4`).
   * Для 3B VLM (LFM2-VL-3B) требуется больше памяти и тензорных ядер, поэтому используется **NVIDIA L4** или A10G.

2. **Масштабируемость**:
   Helm позволяет легко изменять параметр `replicaCount`. Если вы добавите больше камер со склада, GKE Cluster Autoscaler автоматически поднимет новые узлы с GPU, а Deployment создаст новые поды для обработки дополнительных RTSP-потоков.

3. **Темпоральная память**:
   Конфигурации, такие как размер окна памяти (`temporalMemoryFrames`) и детерминированность генерации текста (`temperature`), вынесены в `values.yaml` для гибкой настройки без пересборки Docker-образов.

## Инструкция по запуску

1. **Аутентификация в GCloud**:
   ```bash
   gcloud auth login
   gcloud container clusters get-credentials your-cluster-name --region your-region --project your-project-id
   ```

2. **Подготовка GKE**:
   Убедитесь, что в вашем пуле узлов GKE настроена установка драйверов NVIDIA (в GKE это делается автоматически при создании Node Pool с accelerator).

3. **Установка Helm-чарта**:
   Перейдите в директорию `kubernetes-vlm/additional/` и выполните команду:
   ```bash
   helm upgrade --install warehouse-vlm ./helm --namespace vlm-prod --create-namespace
   ```

4. **Проверка статуса**:
   ```bash
   kubectl get pods -n vlm-prod
   ```
   Убедитесь, что поды перешли в статус `Running`.

## Дальнейшие шаги (Roadmap)
- Настройка `HorizontalPodAutoscaler` (HPA) на основе метрик использования GPU, чтобы отключать дорогостоящие VLM-поды ночью или при отсутствии активности на складе.
- Настройка `External Traffic Policy` и Ingress для приема RTSP/WebRTC потоков от складов (в случае если камеры проксируются в облако).
